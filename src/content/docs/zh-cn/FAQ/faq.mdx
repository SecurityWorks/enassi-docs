## 为什么推荐使用 xrtm 文件?

xrtm 文件的本质是 HTML 文件。我们有以下思考：

尽管 markdown 有许多优点，比如简洁、易于学习、易于阅读等，但它也有一些缺点：

- 兼容性问题：

  - 缺乏统一标准： 没有一个统一的标准，不同的实现可能支持不同的扩展语法，不同的解析器可能对语法的解析存在差异。
  - 方言问题：
    - 存在很多方言， 比如: GitHub Flavored Markdown， GitLab Flavored Markdown， CommonMark， MultiMarkdown， Kramdown 等等。
    - 甚至 Obisidian 和 Typora 等编辑器也会添加特殊的语法。

- 不支持复杂样式： 不支持 CSS 样式，对于需要复杂样式控制的文档来说， markdown 可能不够用。
- 学习曲线： 对于完全不熟悉 markdown 的用户来说，存在一定的难度。

而 HTML 相对 markdown 来说， 除了源代码更复杂之外， 其他方面均比 markdown 更好。
特别是 HTML 有统一的标准，不存在方言问题，任何浏览器都能正常渲染。

我们希望使用有统一标准的，纯文本的数据格式存储文档，保持数据公开透明和高通用性。
我们对比过 markdown / JSON / TOML / HTML 等不同的格式，最终我们决定使用 HTML。

使用 fivim 的编辑器，可以使用类似 markdown 的快捷键实现快速输入。

## 可以不设置密码吗？

不可以， 在**设置向导**里设置的密码有两个用途：

1. 会用于加密您的配置文件，保护您的配置，特别是用于同步的参数。
2. 将会加密您指定的要加密的文件，默认使用 `.enc` 扩展名。

可以使用一个特殊名称的加密文件作为索引文件，这样别人就无法知道叫加密文件的实际文件名。例如:

```
├── secret_directory
│   ├── 100.md.enc
│   ├── 101.md.enc
│   ├── 102.md.enc
│   ├── 200.md.enc
│   ├── 203.md.enc
│   └── 205.md.enc
```

您可以将**200.md.enc**作为索引文件，编辑指向其他文件的一些链接，如：

```
[ssh 密钥](./100.md.enc)
[密码](./101.md.enc)
[电子邮件](./102.md.enc)
[银行](./203.md.enc)
[家族](./205.md.enc)
```

## 如何更改密码？

首先，密码有两个用途:解锁登录屏幕和加密用户要求加密的文件。

因此，如果您想更改密码，可以按照以下步骤操作：

1. 导出所有已加密的文件。
2. 删除您的应用程序数据，包括配置文件和用户数据。
3. 重新打开您的应用程序，在**设置向导**页面中设置新密码。
4. 新建使用加密文件扩展名(默认为**.enc**)的文件，将文件内容复制到新文件中并保存。

## 当同步服务不可用时，我无法编辑文件。

因为制作一个完美的同步服务并不容易。因此，如果您设置了同步服务，所有更改将首先发送到远程服务器，在同步服务返回成功后，本地文件将更改。这将确保文件是正确的。

## 为什么不使用 git 同步?

如果出现文件内容冲突，普通用户解决起来比较麻烦， 为了让普通用户用起来更简单。

## 我可以使用 git 吗？

如果你想使用 **git** 或者其它版本管控工具进行同步，需要禁用 fivim 的同步，因为 **从远程初始化** 会删除 **~/fivim/user-files** 目录，这将会删除 **.git** 目录造成错误。
